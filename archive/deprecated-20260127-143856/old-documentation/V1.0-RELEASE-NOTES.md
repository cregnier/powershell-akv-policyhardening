## üöÄ Quick Start - 3 Steps to Deploy

### Step 1: Extract & Prepare (5 minutes)
```powershell
# Extract ZIP to working directory
Expand-Archive -Path AzureKeyVaultPolicyGovernance-v1.0.zip -DestinationPath C:\AKV-Governance

# Navigate to folder
cd C:\AKV-Governance

# Install prerequisites
Install-Module -Name Az.Accounts, Az.Resources, Az.PolicyInsights, Az.KeyVault -Force
```

### Step 2: Setup Infrastructure (10 minutes)
```powershell
# Connect to Azure
Connect-AzAccount
Set-AzContext -Subscription "<your-subscription-id>"

# Create required infrastructure
.\Setup-AzureKeyVaultPolicyEnvironment.ps1 -CreateTestEnvironment
```

### Step 3: Deploy Policies (5 minutes)
```powershell
# OPTION A: DevTest (safe testing - 30 policies)
.\AzPolicyImplScript.ps1 -DeployDevTest -SkipRBACCheck

# OPTION B: Production Audit (monitoring - 46 policies)
.\AzPolicyImplScript.ps1 -DeployProduction -SkipRBACCheck

# Check compliance
.\AzPolicyImplScript.ps1 -CheckCompliance -TriggerScan -SkipRBACCheck
```

## üîß Troubleshooting

### Common Issues

**"Module not found"**
```powershell
Install-Module -Name Az.Accounts, Az.Resources, Az.PolicyInsights -Force
```

**"Not connected to Azure"**
```powershell
Connect-AzAccount
Set-AzContext -Subscription "<subscription-id>"
```

**"Insufficient permissions"**
- Requires: Policy Contributor or Owner role
- Check: `Get-AzRoleAssignment -SignInName "<your-email>"`

**"No compliance data"**
- Wait 15-30 minutes for Azure Policy evaluation
- Use `-TriggerScan` flag to start evaluation
